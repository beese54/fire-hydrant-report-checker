# SCDF Hydrant Form Checking

A password-protected Streamlit app for SCDF officers to vet hydrant inspection submissions and contractor rectification work. Deployed as an Airbase app.

---

## Overview

This app supports two workflows: verifying hydrant defect reports submitted by inspectors via FormSG, and verifying rectification work subsequently carried out by contractors. Officers upload an Excel submission log (managed via Plumber on Microsoft 365) alongside email attachments from FormSG, vet each entry visually, and generate structured Excel reports for downstream action.

---

## System Workflow

### Phase 1 — Hydrant Inspection Reporting

1. Inspector submits a hydrant inspection report via **FormSG**, including defect details and standardised photos.
2. **Plumber** (OGP) captures each submission and appends it to an Excel sheet stored on **Microsoft 365**, which serves as the central database.
3. Plumber's **logic function** filters the Excel sheet to surface only entries where hydrants require rectification.
4. Officer downloads the filtered Excel sheet and the email attachments generated by FormSG (`.eml` files).

---

### Phase 2 — Officer Vetting (`SCDF Form Submission Checks` tab)

5. Officer uploads the filtered Excel log and the FormSG `.eml` attachments into the app.
6. The app matches each `.eml` to its corresponding Excel row using the **Submission ID**, then displays the hydrant details and defect photos for each entry.
7. Officer reviews each entry and marks it as **"Requires Rectification"** or **"Does Not Require Rectification"**.
8. The app generates a **rectification Excel report** listing all entries that require attention.
9. Officer emails the report to the **contractor** as an instruction to carry out the rectification works.

---

### Phase 3 — Contractor Rectification Submission

10. Once works are completed, the contractor submits a separate **FormSG** with photos of the rectified hydrant taken from standardised angles (front, left side, right side, back).
11. Officer downloads the updated **Microsoft 365 Excel** (which now includes contractor submission rows) and the corresponding FormSG `.eml` attachments.

---

### Phase 4 — Contractor Work Vetting (`Contractor Rectification Checks` tab)

12. Officer uploads the contractor Excel and `.eml` attachments into the app.
13. The app performs a **dual-key match** (Submission ID + Hydrant Number) to pair each email to its Excel row, then displays the contractor's photos alongside the original defect details.
14. Officer vets each entry and marks it as **"Satisfactory"** or **"Not Satisfactory"**. A mandatory reason must be provided for every not-satisfactory entry.
15. The app generates two separate reports:
    - **Satisfactory report** — compiled (all entries in one file) or separated by fire station (one file per station).
    - **Not Satisfactory report** — includes the officer's reasons; sent back to the contractor for re-work.
16. Officer attaches the satisfactory report to an email to **SCDF**, confirming that hydrant rectification is complete.

---

## Features

- Password-protected access
- Automatic matching of FormSG `.eml` attachments to Excel submission logs
- Side-by-side photo display for visual vetting
- Dual-tab interface separating inspection vetting from contractor vetting
- Satisfactory reports can be compiled or split by fire station
- Not-satisfactory reports include mandatory officer reasons
- All reports exported as `.xlsx` with embedded photos

## Getting Started

**Prerequisites:** Docker and Docker Compose

```bash
# 1. Set the app password via environment variable
export APP_PASSWORD=your_password_here

# 2. Build and start
docker compose up --build

# 3. Open in browser
#    http://localhost:8501
```

To stop the app:
```bash
docker compose down
```

---

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `APP_PASSWORD` | *(must be set)* | Login password — configure before running |
| `MAX_UPLOAD_MB` | `20` | Maximum total size of `.eml` uploads in MB |

## Future Work

> The FormSG inspection forms were designed to require submitters to upload standardised photos of the hydrant from four fixed angles — front, left side, right side, and back. Beyond their immediate use for manual officer vetting, these photos constitute a structured, consistently labelled image dataset. This lays the foundation for future exploration of **computer vision models** that could automatically detect hydrant defects from photos, potentially reducing manual review time and improving consistency across inspections.
